## מדיניות PDF (קריטי)

- אין לייבא/להעתיק PDFs אוטומטית מה-Downloads או מכל מקום אחר.
- רק קבצים שיוכנסו ידנית לנתיב `pdf/` בתוך הריפו ייכללו בספר.
- כל שינוי בתכולת `pdf/` חייב להיות החלטה מפורשת ולבוא עם commit.

# דרישות מערכת (כלליות) — מצטבר

## Requirements

- לבנות פלטפורמת ספר דיגיטלי להצגת קבצי PDF מתוך הריפו.
- מקור האמת של הפרקים הוא התיקייה `pdf/` (מותר מבנה תיקיות פנימי; כל PDF = פרק/יחידה).
- המערכת מתעדכנת אוטומטית כאשר מוסיפים PDFs:
  - סריקה/זיהוי
  - עדכון תוכן עניינים
  - עדכון ניווט
  - פריסה מחדש ל־GitHub Pages
- קישור ציבורי קבוע (GitHub Pages): `https://<owner>.github.io/digitalbook/`.
- ממשק בעברית בלבד + RTL.
- ניווט:
  - עמוד קודם/עמוד הבא
  - פרק קודם/פרק הבא
- שני מנגנוני תוכן עניינים:
  - תוכן עניינים גלובלי לכל הפרקים (מבוסס עץ תיקיות תחת `pdf/`)
  - תוכן עניינים פנימי בתוך הפרק (מבוסס Outline של ה־PDF כשזמין)
- חיפוש:
  - חיפוש לפי שם פרק/תיקייה בתוכן העניינים
  - חיפוש בתוך הפרק לפי טקסט כאשר ניתן לחלץ טקסט מה־PDF
- PWA:
  - manifest
  - אייקונים
  - אפשרות התקנה (Add to Home Screen)

## Architecture decisions

- אתר סטטי תחת `site/` שנפרס ל־GitHub Pages.
- יצירת תוכן עניינים גלובלי ב־CI באמצעות `site/scripts/generate-toc.mjs`:
  - סריקת עץ `pdf/`
  - יצירת `site/generated/toc.json`
  - העתקת PDFs ל־`site/pdf/` כדי ש־Pages יגיש אותם.
- רינדור PDF בדפדפן באמצעות PDF.js.
- תוכן עניינים פנימי (Outline) וחיפוש בתוך פרק נוצרים בזמן ריצה (best-effort) כדי להימנע מעיבוד כבד ב־CI.

## Completed features

- UI בעברית בלבד + RTL.
- תוכן עניינים גלובלי (עץ תיקיות) עם חיפוש לפי שם.
- ניווט עמודים + ניווט פרקים.
- תוכן עניינים פנימי לפרק (PDF Outline) כאשר זמין.
- חיפוש בתוך הפרק (כאשר יש טקסט לחילוץ; זיהוי PDF סרוק/ללא טקסט).
- PWA: manifest + icons + רישום Service Worker.
- GitHub Actions לפריסה אוטומטית ל־GitHub Pages.
