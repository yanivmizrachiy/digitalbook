# RULES.md — digitalbook (Single Source of Truth)

## עקרונות ברזל
- הריפו `digitalbook` הוא מקור אמת יחיד.
- כל שינוי חייב commit+push.
- כל שינוי חייב לעדכן את RULES.md באותו commit.
- דרישות מצטברות: לא מוחקים ולא משמיטים דרישות.
- אין סתירות: אם דרישה חדשה מתנגשת — עוצרים ומיישבים.

## מטרת המוצר
ספר דיגיטלי ציבורי המבוסס על קבצי PDF שמאוחסנים בריפו, עם ניווט נוח, תוכן עניינים מתעדכן, וממשק פרימיום בעברית RTL.

## דרישות מחייבות (כולן)
- קישור ציבורי קבוע לספר (GitHub Pages).
- PDFs יישמרו בתיקייה `pdf/`.
- הוספה/עדכון PDFs תעדכן אוטומטית:
  - תוכן עניינים
  - ניווט פרקים
  - פריסה ל-Pages
- ממשק בעברית בלבד + RTL מלא.
- כפתורי ניווט בעברית, עובדים:
  - תוכן עניינים
  - פרק קודם / פרק הבא
  - עמוד קודם / עמוד הבא
- תוכן עניינים פנימי שמתעדכן אוטומטית.
- חיפוש בסיסי לפחות לפי שם פרק (ובעתיד הרחבה).
- חוויית קריאה נוחה (מובייל/טאבלט/דסקטופ), קריא, לא צפוף, “פרימיום”.
- PWA: אייקון קבוע למסך הבית/שולחן העבודה, manifest + icons + service worker.
- אוטומציה מלאה: GitHub Actions בונה TOC ומפרסם ל-GitHub Pages בכל push.
- תצוגה מקדימה נוחה לפיתוח (הנחיות ב-README).
